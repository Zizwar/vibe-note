# تقرير تنفيذ الإصدار الثاني (v2.0.0) — Vibe Note

> **التاريخ:** 18 يوليو 2026 · **الفرع:** `claude/vibe-note-v2-upgrade-jze3i0` · **الأساس:** v1.5 (Expo 56)
>
> هذه الجولة تنفّذ قفزة v2: دردشة احترافية بجلسات محفوظة، نظام أنواع (برومبت/نوت/سياق)، سياقات قابلة لإعادة الاستخدام تُحقن في المحادثات، تركيب البرومبتات (سلاسل "الخطوة التالية")، مزوّدان جديدان (Anthropic Claude وOllama المحلي)، هجرات قاعدة بيانات مُرقَّمة، وتوصيل بحث FTS5 فعلياً.

---

## 1. ما الجديد للمستخدم

### 1.1 دردشة احترافية بجلسات محفوظة (UP-13 + UP-14)
- **المحادثات تُحفظ الآن في قاعدة البيانات** (`chat_sessions` + `chat_messages`) — أغلق التطبيق وارجع، محادثاتك موجودة.
- **جلسات متعددة:** زر "المحادثات" يعرض كل الجلسات (تثبيت، حذف، فتح)، وزر "+" يبدأ محادثة جديدة. عنوان الجلسة يتولّد تلقائياً من أول رسالة.
- **إجراءات على كل ردّ:** نسخ، **حفظ كنوت** (فوري)، **حفظ كبرومبت** (يفتح شاشة الإنشاء معبّأة)، و**إعادة توليد** آخر ردّ.
- **الضغط المطوّل على أي فقاعة ينسخها.**
- عرض اسم المزوّد والنموذج النشطَين في شريط المساعد.

### 1.2 السياقات (Contexts) — كتل سياق قابلة لإعادة الاستخدام
- نوع جديد من العناصر: **سياق** — مثلاً "هوية العلامة التجارية" أو "شخصية مطوّر خبير".
- **إرفاق سياقات بالمحادثة:** زر الطبقات في شريط المساعد يفتح منتقي السياقات؛ المرفَق يُحقن كـ system context في كل طلب.
- **إرفاق سياقات ببرومبت:** من شاشة تفاصيل البرومبت؛ عند تشغيل البرومبت في الدردشة تُحمَّل سياقاته تلقائياً في الجلسة.

### 1.3 تركيب البرومبتات (Prompt Composition / Chains) — UP-17
- **ربط برومبت ببرومبتات أخرى** من شاشة التفاصيل (قسم "برومبتات مرتبطة" + منتقي).
- بعد تشغيل برومبت في الدردشة، تظهر برومبتاته المرتبطة كرقائق **"الخطوة التالية"** فوق صندوق الإدخال — نقرة واحدة تُشغّل التالي في نفس المحادثة (مخرج كل خطوة يبقى في سياق المحادثة فيتغذّى عليه التالي).
- حذف برومبت يُنظّف الإشارات المعلّقة إليه من قوائم الربط والسياقات.

### 1.4 تشغيل برومبت بالمتغيّرات داخل الدردشة
- زر ⚡ في صف الإدخال يفتح لائحة البرومبتات؛ اختيار برومبت **بمتغيّرات** يعرض **نموذج ملء داخلي** (حقول نصية + خيارات) قبل الإرسال، مع تسجيل الاستخدام والسجل.

### 1.5 نظام الأنواع: برومبت / نوت / سياق
- محدد نوع في شاشة الإنشاء (يُخفي المنصّة وتلميح المتغيّرات لغير البرومبت).
- شريط تصفية في الرئيسية: الكل / برومبتات / نوتات / سياقات.
- شارات نوع مميّزة بالألوان في البطاقات وشاشة التفاصيل.

### 1.6 مزوّدون جدد (UP-15)
- **Anthropic Claude** أصلي (`/v1/messages` مع `x-api-key`).
- **Ollama المحلي** (متوافق مع OpenAI على `localhost:11434`، **بدون مفتاح**) — يعزّز فلسفة الخصوصية.
- تُدمج المزوّدات الجديدة تلقائياً لدى المستخدمين القدامى عند التحميل.

## 2. تحت الغطاء (Technical)

| المجال | التغيير |
|---|---|
| الهجرات (UP-04) | `PRAGMA user_version` + مصفوفة هجرات مرقّمة idempotent (`migrateToV1`, `migrateToV2`) بدل `try/catch ALTER` المتفرّقة |
| المخطط v2 | أعمدة `kind`, `linked_ids`, `context_ids` على `prompts` + جدولا `chat_sessions` و`chat_messages` بفهرس ومفتاح أجنبي CASCADE |
| البحث (UP-06) | `getAllPrompts` يستخدم **FTS5 فعلياً** مع ترتيب `bm25()` وبادئات مطابقة لكل كلمة، وسقوط آمن إلى LIKE |
| محرّك الذكاء | `callAIChat(messages)` موحّد: Gemini بـ `systemInstruction` وأدوار `user/model` صحيحة (بدل تسطيح المحادثة نصاً)، Anthropic أصلي، OpenAI-متوافق؛ رسائل خطأ تتضمّن كود الحالة ومقتطف الجسم |
| حقن السياق | `buildChatSystemPrompt(contexts)` يبني system prompt واحداً بشخصية Vibe + كتل السياق المرفقة |
| مخزن جديد | `chatStore` (zustand): جلسات/رسائل مدعومة بالقاعدة، إرسال، إعادة توليد، إدارة سياقات الجلسة |
| الأنواع | `ItemKind`, `ChatSession`, `ChatMessage` في `types`؛ `VibeNote` اكتسب `kind/linkedIds/contextIds` |
| التصدير/الاستيراد | صيغة `.vibe` v2 تحمل النوع والروابط؛ الاستيراد يقبل ملفات v1 القديمة بأمان |
| البذور | سياقان جاهزان + نوت + سلسلة ربط جاهزة (Article Writer ← Marketing Copy / Email) لإظهار الميزات فوراً |
| i18n | كل المفاتيح الجديدة بالعربية والإنجليزية والفرنسية مع دعم RTL كامل |

**ملفات جديدة:** `src/database/chatQueries.ts`, `src/stores/chatStore.ts`, `docs/v2-implementation-report.md`
**ملفات معدّلة رئيسية:** `schema.ts`, `queries.ts`, `aiService.ts`, `AIAssistantScreen.tsx` (إعادة بناء الدردشة), `PromptDetailScreen.tsx`, `CreatePromptScreen.tsx`, `HomeScreen.tsx`, `PromptCard.tsx`, `VariableFiller.tsx`, `settingsStore.ts`, `promptStore.ts`, `historyStore.ts`, `seed.ts`, `importExport.ts`, `App.tsx`, `strings.ts`, `README.md`

## 3. التحقق والمراجعة العدائية

- `npx tsc --noEmit` → **نظيف (0 أخطاء)**.
- `npx expo-doctor` → **19/21**؛ الفحصان الفاشلان (مخطط Expo وRN Directory) يتطلبان شبكة خارجية محجوبة في بيئة البناء — ليسا مشكلة في المشروع.
- ترقية قواعد بيانات المستخدمين القدامى مغطاة: `user_version=0` يعيد تشغيل هجرة v1 (idempotent) ثم يطبّق v2.
- **مراجعة عدائية متعددة الوكلاء** (24 وكيلاً: 5 أبعاد مراجعة × تحقق مستقل لكل ملاحظة) على كامل الفرق، وأُصلحت كل النتائج المؤكدة:
  1. إعادة تشغيل البرومبت المزروع عند العودة من "حفظ كبرومبت" → حارس `seedNonce` على مستوى الوحدة.
  2. `regenerateLast` كان يحذف الردّ القديم قبل الطلب → الحذف صار بعد نجاح الردّ الجديد فقط، مع استرجاع عند الفشل.
  3. إسقاط صامت للإرسال أثناء ردٍّ جارٍ (مع تسجيل استخدام زائف) → تنبيه "انتظر اكتمال الرد الحالي" قبل أي أثر جانبي.
  4. تفعيل Ollama بلا مفتاح كان محجوباً في شاشة الإعدادات → مسموح لمزوّد `requiresKey: false`، مع بطاقات معلومات ونماذج لـ Anthropic وOllama وحقل Base URL قابل للتحرير (مهم للأجهزة الفعلية: `10.0.2.2` لمحاكي أندرويد).
  5. بحث FTS بلا نتائج كان يُخفي مطابقات وسط الكلمة → سقوط تلقائي إلى LIKE عند صفر نتائج.
  6. `addColumn` كان يبتلع كل أخطاء الهجرة → يتجاهل "duplicate column" فقط ويُفشل الهجرة عند غيره كي لا يُختم الإصدار فوق مخطط ناقص.
  7. حذف عنصر لم يكن ينظّف `context_ids` في جلسات الدردشة → تنظيف شامل.
  8. تراكم جلسات "Chat" فارغة من زر "محادثة جديدة" → الجلسة تُنشأ الآن بكسل (lazy) مع أول رسالة.
  9. تسرّب مصفاة النوع من الرئيسية إلى شاشة المفضلة → المفضلة تستعلم القاعدة مباشرة.
  10. حذف الجلسة أثناء ردٍّ جارٍ كان يكسر المفتاح الأجنبي → فحص وجود الجلسة قبل إدراج الردّ.
  11. فتح نموذج الملء من رقاقة سلسلة ثم الرجوع كان يعرض لائحة فارغة → تحميل اللائحة عند الحاجة.

## 4. ما بعد v2 (مقترح v2.1+)

- بثّ الردود (streaming) عند توفّر دعم SSE مستقر في RN/Expo fetch.
- مقارنة A/B بين مزوّدين لنفس البرومبت.
- تتبّع التوكنز/التكلفة الفعلية لكل جلسة.
- تفعيل البرومبت الصوتي (`audio_base64`) — UP-18.
- المجلّدات (`folder_id`) والإجراءات المجمّعة — الموجة B.
