# Proomy Note — وثيقة رؤية المنتج الشاملة

> **"البرومبت هو لغة البرمجة الجديدة"**
> تطبيق أندرويد لإدارة، تنظيم، واستخدام البرومبتات بذكاء — محلياً أولاً، مجتمعياً لاحقاً.

---

## 1. الفجوة في السوق (Gap Analysis)

### المشكلة الحقيقية

كل يوم، ملايين المستخدمين يكتشفون برومبتات ممتازة من مصادر متعددة:

- تغريدات على X (تويتر) — "هذا البرومبت غيّر حياتي مع Midjourney!"
- فيديوهات يوتيوب — "أفضل 10 برومبتات لـ ChatGPT"
- قروبات تيليقرام وواتساب — "جربوا هذا البرومبت للكتابة"
- ريديت ومنتديات — برومبتات متقدمة للمبرمجين
- تجارب شخصية — برومبتات طوّرها المستخدم بنفسه

**أين تروح هذي البرومبتات؟**

| الطريقة الحالية | المشكلة |
|---|---|
| نوتات الجوال | فوضى بدون تصنيف، ما تلقاها وقت الحاجة |
| بوكماركات المتصفح | مدفونة بين مئات الروابط |
| سكرينشوتات | ما تقدر تنسخها، تشغل مساحة |
| محادثات واتساب/تيليقرام | تضيع مع الوقت |
| ملفات Google Docs | ثقيلة وبطيئة للاستخدام السريع |
| الذاكرة البشرية | "وش كان البرومبت اللي استخدمته ذاك اليوم؟!" |

### المنافسون الحاليون ونقاط ضعفهم

| المنافس | نوعه | الضعف الرئيسي |
|---|---|---|
| PromptBase | ماركت بليس (ويب) | تركيزه على البيع مو على الاستخدام اليومي، ما عنده تطبيق |
| FlowGPT | مجتمع برومبتات (ويب) | معقد، كثير تشتت، ما فيه تنظيم شخصي |
| AIPRM (إضافة كروم) | إضافة لـ ChatGPT فقط | محدود بمنصة واحدة، ما يشتغل مع Midjourney أو Claude |
| ملاحظات الجوال | تطبيقات عامة | ما صُمّمت للبرومبتات — بدون متغيرات، بدون تصنيف بالأداة |

### الفرصة

**ما في تطبيق موبايل واحد مصمم خصيصاً لإدارة البرومبتات بذكاء.**
Proomy Note يملأ هذي الفجوة بالضبط.

## 2. رؤية المنتج (Product Vision)

### الهوية

| العنصر | التفصيل |
|---|---|
| **الاسم** | Proomy Note (بروومي نوت) |
| **الشعار** | "مكتبتك الذكية للبرومبتات" |
| **الجمهور** | كل من يستخدم أدوات AI يومياً — مسوقين، مبرمجين، مصممين، صنّاع محتوى، طلاب |
| **المنصة** | Android (React Native) — ثم iOS |
| **اللغة** | عربي + إنجليزي (RTL/LTR) |

### الجملة الوصفية (Elevator Pitch)

> Proomy Note هو التطبيق الوحيد اللي يخليك تحفظ برومبتاتك، تصنّفها حسب الأداة والمجال، تعبّي المتغيرات بنقرات، وتنسخ النتيجة جاهزة — كل هذا أسرع من فتح ملف ملاحظات.

## 3. الميزات التنافسية الأساسية (Killer Features)

### 3.1 المتغيرات الديناميكية (Dynamic Variables) ⭐ الميزة الأقوى

هذي الميزة اللي تفرّق Proomy Note عن أي تطبيق ملاحظات عادي.

**كيف تشتغل:**

المستخدم يحفظ البرومبت بهذا الشكل:

```
صمم صورة لشخصية {{character}} في بيئة {{environment}} بأسلوب {{art_style}}
```

عند الاستخدام:
1. التطبيق يكشف المتغيرات تلقائياً: `character`, `environment`, `art_style`
2. يعرض حقول إدخال سريعة (Bottom Sheet)
3. المستخدم يعبّي الحقول
4. ينسخ البرومبت النهائي الجاهز بضغطة

**المستوى المتقدم:**

- **قيم افتراضية:** `{{art_style|واقعي}}` — القيمة "واقعي" تكون معبّأة مسبقاً
- **قوائم اختيار:** `{{art_style:واقعي|كرتوني|بكسل|أنيمي}}` — يظهر Dropdown بدل حقل نصي
- **سجل القيم:** التطبيق يحفظ آخر القيم المستخدمة ويقترحها (autocomplete)
- **مجموعات القيم (Presets):** المستخدم يحفظ مجموعة قيم باسم (مثلاً: "إعدادات بورتريه") ويعيد تحميلها

### 3.2 التصنيف المزدوج (Dual Classification)

**البعد الأول — المجال:**
🎨 توليد صور · 🎬 فيديو · ⌨️ برمجة · ✍️ كتابة · 📢 تسويق · 💼 أعمال · 🎵 موسيقى · 🎓 تعليم · 📦 أخرى

**البعد الثاني — الأداة/المنصة:**
ChatGPT · Claude · Gemini · Midjourney · DALL-E · Stable Diffusion · Sora · Runway · Kling · Copilot · Cursor · v0 · Lovable · أخرى

**البعد الثالث — وسوم حرة (Tags):**
المستخدم يضيف وسوم مخصصة: `#portrait` `#arabic` `#seo` `#react`

هذا يعني المستخدم يقدر يبحث بطرق متعددة:
- "أبي برومبتات تسويق لـ ChatGPT"
- "كل برومبتات Midjourney المفضلة"
- "أي شي فيه تاق #seo"

### 3.3 النسخ الذكي (Smart Copy)

- **نسخ خام:** ينسخ البرومبت كما هو
- **نسخ مع تعبئة:** يعرض حقول المتغيرات → يعبّي → ينسخ الناتج
- **فتح مباشر:** زر يفتح ChatGPT/Claude مع البرومبت محمّل (Deep Link / Share Intent)
- **ويدجت الشاشة الرئيسية (Home Widget):** أيقونة سريعة للبرومبتات المفضلة — نقرة واحدة = نسخ

### 3.4 استيراد ذكي (Smart Import)

- **من النص:** المستخدم ينسخ برومبت من أي مكان → يفتح Proomy → يضيفه مع اقتراح تصنيف تلقائي
- **من الشير (Share Intent):** المستخدم يشارك نص من أي تطبيق → Proomy يستقبله كبرومبت جديد
- **من الصور (OCR):** المستخدم يصور سكرينشوت لبرومبت → التطبيق يستخرج النص
- **من الملفات:** استيراد ملف JSON/CSV لمجموعة برومبتات

### 3.5 مجلدات ومجموعات (Folders & Collections)

- **مجلدات:** تنظيم هرمي (مثلاً: عمل > تسويق > محتوى سوشال ميديا)
- **مجموعات ذكية (Smart Collections):** تُحدّث تلقائياً بناءً على شروط
  - مثال: "كل برومبتات Midjourney المضافة هذا الشهر"
  - مثال: "برومبتات استخدمتها أكثر من 10 مرات"

### 3.6 تاريخ الاستخدام والإحصائيات

- **عداد استخدام** لكل برومبت
- **آخر استخدام** مع التاريخ
- **تاريخ التعبئات:** يحفظ كل مرة عبّيت المتغيرات — تقدر ترجع لنسخة سابقة
- **إحصائيات عامة:** أكثر برومبت استخداماً، أكثر أداة تستخدمها، نشاطك الأسبوعي

### 4.2 محرك المتغيرات (Variable Parser Engine)

```typescript
// استخراج المتغيرات من النص
function extractVariables(content: string): VariableDefinition[] {
  const regex = /\{\{(\w+)(?:\|([^}]+))?\}\}/g;
  const variables: VariableDefinition[] = [];
  const seen = new Set<string>();
  
  let match;
  while ((match = regex.exec(content)) !== null) {
    const name = match[1];
    if (seen.has(name)) continue;
    seen.add(name);
    
    const defaultOrOptions = match[2];
    
    // تحديد النوع: هل فيه خيارات (قائمة) أو قيمة افتراضية؟
    if (defaultOrOptions?.includes(':')) {
      // {{var:opt1|opt2|opt3}} → select
      const options = defaultOrOptions.split(':')[1]?.split('|') || [];
      variables.push({
        name,
        type: 'select',
        options,
        defaultValue: options[0],
        recentValues: [],
      });
    } else {
      variables.push({
        name,
        type: 'text',
        defaultValue: defaultOrOptions || undefined,
        recentValues: [],
      });
    }
  }
  return variables;
}

// بناء البرومبت النهائي
function buildFinalPrompt(
  content: string, 
  values: Record<string, string>
): string {
  let result = content;
  for (const [key, value] of Object.entries(values)) {
    result = result.replace(
      new RegExp(`\\{\\{${key}(?:\\|[^}]*)?\\}\\}`, 'g'), 
      value
    );
  }
  return result;
}
```

## 5. الهيكل التقني (Technical Architecture)

### 5.1 Stack التقني

| الطبقة | التقنية | السبب |
|---|---|---|
| **Framework** | React Native (Expo) | تطوير سريع + Android أولاً + iOS لاحقاً |
| **Navigation** | Expo Router | file-based routing، أبسط وأحدث |
| **التخزين المحلي** | WatermelonDB | قاعدة بيانات SQLite محسّنة لـ React Native، سريعة جداً مع آلاف السجلات، تدعم المزامنة |
| **الحالة (State)** | Zustand | خفيفة وبسيطة وتتكامل مع WatermelonDB |
| **التصميم** | NativeWind (Tailwind) | utility-first يدعم RTL |
| **التصنيف التلقائي** | on-device ML (اختياري) | لاقتراح التصنيف عند إضافة برومبت |
| **OCR** | ML Kit (Google) | استخراج نص من صور/سكرينشوتات |
| **المزامنة لاحقاً** | Supabase | Auth + Database + Realtime + Storage |

### 6.2 التفاعلات الذكية

- **سحب يمين على البطاقة** → نسخ سريع
- **سحب يسار** → مفضلة / إلغاء مفضلة
- **ضغط مطوّل** → قائمة (تعديل، حذف، مشاركة، نقل لمجلد)
- **شير من أي تطبيق** → Proomy يستقبل ويعرض "إضافة كبرومبت؟"

## 7. أفكار إبداعية إضافية (Beyond the Basics)

### 7.1 وضع التدريب (Practice Mode)

شاشة يختار فيها المستخدم برومبت → يعبّي المتغيرات → يرسل مباشرة لـ AI API → يشوف النتيجة داخل التطبيق — بدون ما يطلع من Proomy. (يتطلب API Key من المستخدم.)

### 7.2 سلاسل البرومبتات (Prompt Chains)

بعض المهام تحتاج أكثر من برومبت متسلسل:
```
سلسلة: كتابة مقال احترافي
├── الخطوة 1: توليد عناوين (برومبت عناوين)
├── الخطوة 2: كتابة مسودة (برومبت كتابة)
├── الخطوة 3: تحسين SEO (برومبت SEO)
└── الخطوة 4: كتابة وصف ميتا (برومبت ميتا)
```

المستخدم يبني "سلسلة" ويشغّلها خطوة بخطوة.

### 7.3 نسخ البرومبت المعدّل (Version History)

كل تعديل على البرومبت يُحفظ كنسخة. المستخدم يقدر يقارن بين النسخ ويرجع لأي نسخة سابقة. هذا مهم لأن المستخدمين يعدّلون البرومبتات باستمرار لتحسينها.

### 7.4 البرومبت المشروح (Annotated Prompt)

المستخدم يقدر يضيف "شروحات" على أجزاء من البرومبت — ليش كتبت "35mm film"؟ ليش "shallow depth of field"؟ هذا يساعد في:
- تعلّم كتابة البرومبتات
- مشاركة البرومبت مع شرح للمجتمع
- تذكّر السبب وراء كل جزء

### 7.5 بطاقات الإلهام (Inspiration Cards)

شاشة يومية تعرض "برومبت اليوم" — برومبت مميز من مجتمع Proomy أو من مكتبة مُعدّة مسبقاً. تشجّع الاستخدام اليومي.

### 7.6 تصنيف تلقائي بالذكاء الاصطناعي

عند إضافة برومبت جديد، التطبيق يقترح:
- التصنيف المناسب (صور؟ كود؟ كتابة؟)
- الأداة المناسبة (يتعرّف على أنماط Midjourney vs ChatGPT)
- وسوم مقترحة

يمكن تنفيذه محلياً بقواعد regex بسيطة في البداية (مثلاً: إذا فيه "photo" أو "--ar" → Midjourney).

### 7.7 قوالب (Templates)

التطبيق يأتي مع مكتبة قوالب جاهزة مصنّفة:
- "باقة المسوّق" — 20 برومبت تسويقي
- "باقة المبرمج" — 15 برومبت للبرمجة
- "باقة المصمم" — 25 برومبت لتوليد الصور

المستخدم يستورد الباقة بنقرة.

### 7.8 الويدجت والاختصارات (Widgets & Quick Actions)

- **ويدجت صغير (2x1):** يعرض برومبت واحد مثبّت مع زر نسخ
- **ويدجت متوسط (4x2):** يعرض آخر 3 برومبتات مستخدمة
- **ويدجت كبير (4x4):** يعرض المفضلة + بحث سريع
- **Quick Tile:** في لوحة الإشعارات — يفتح إضافة برومبت سريع

## 11. تحليل التقنيات البديلة

### لماذا WatermelonDB وليس MMKV أو AsyncStorage؟

| المعيار | AsyncStorage | MMKV | WatermelonDB |
|---|---|---|---|
| السرعة | بطيء | سريع جداً | سريع جداً |
| الاستعلامات المعقدة | لا | لا | نعم (SQL) |
| البحث النصي (FTS) | لا | لا | نعم |
| المزامنة السحابية | يدوي | يدوي | مدمج (Sync) |
| عدد السجلات | محدود | جيد | ممتاز (100K+) |
| العلاقات بين البيانات | لا | لا | نعم |

**القرار:** WatermelonDB لأنها تدعم استعلامات SQL المعقدة (نحتاجها للبحث والفلترة والمجموعات الذكية) + لها sync adapter جاهز لـ Supabase.

### لماذا Expo وليس bare React Native؟

- **سرعة التطوير:** expo-router + EAS Build + OTA Updates
- **الويدجت:** مدعوم عبر `expo-widget`
- **OCR:** `expo-camera` + ML Kit
- **Share Intent:** مدعوم
- **إذا احتجنا native module:** Expo Dev Client يحل المشكلة
